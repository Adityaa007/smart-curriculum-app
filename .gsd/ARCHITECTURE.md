## Components

### Frontend (React)
- **Purpose:** User interface for teachers and students.
- **Location:** `frontend/src/`
- **Key Modules:**
  - `pages/`: Role-specific dashboards (Attendance, Timetable, Routine).
  - `lib/`: Standardized utilities (date, firestore, icons).
  - `contexts/`: Authentication state management.
  - `components/ui/`: Premium, glassmorphic UI components.

### Backend (Flask)
- **Purpose:** Handles computationally expensive or proprietary logic (GenUI, complex aggregations).
- **Location:** `backend/`
- **Dependencies:** `firebase-admin`, `google-generativeai`.

## Data Flow

1. **Authentication:** Users sign in via Firebase Auth in the frontend.
2. **Real-time Sync:** Dashboards listen to Firestore collections (`routines`, `attendance`, `timetable`) using `onSnapshot`.
3. **AI Generation:** Frontend sends user profile and timetable data to the Flask `/api/generate-routine` endpoint, which returns a structured JSON routine.
4. **Attendance Logic:**
   - **QR Mode:** Teacher generates a session; students scan via scan page.
   - **Face Mode:** Face descriptors are matched on the client-side (face-api.js) and validated against Firestore.
